PASCAL on the HP Workstation

1. pc Compiler

The pc command compiles Pascal source files. If the "-C" or "-c" option is not specified (see below) pc compiles and loads the source code and generates an executable file. The format of the pc command line is:

pc [options] file

The more common compiler options are discussed below. If particular options are not specifed, pc uses a set of defaults. Options may be listed in any order following the pc command. The Pascal source code file name must end in ".p".

2. pc Options List

Here is a partial list of some of the more commonly used pc options: 
   -A          Produce warnings for the use of non-ANSI-Pascal
   -o<name>    Specifies the name of the object file as <name>.
   -C          For syntax/semantic checking only.
   -c          Requests that object files not be sent to the link editor.
   -O          Requests code optimization during compilation.
   -I<dir>     Specifies the search path for include files.
   -g          Produces information for the debugger.
   -p          Sets up the object file for profiling with prof.
   -v          Displays language processing commands as they are invoked.
   -L          Write a program listing to stdout.
   file.p      File containing PASCAL source code file.
   file.o      File containing object code.
For more on the available options of the pc compiler type "man pc" at the UNIX prompt. For details, specifics and system dependencies, one can refer to the manuals.

3. Program Execution

The simplest method of program execution is the "compile, load and go" sequence. Here is a Pascal example:

pc source.p
a.out

In these example, the pc commands compiles and loads the programs "source.p" and automatically generate the executable file "a.out". To execute "a.out", simply type the word "a.out" at the UNIX prompt. Your program is then off-and-running.
