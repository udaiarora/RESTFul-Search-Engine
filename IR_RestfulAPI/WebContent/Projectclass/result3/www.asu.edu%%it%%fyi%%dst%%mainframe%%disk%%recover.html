Recovering Disk Files from System Backup Facility

IMPORTANT:

You are responsible for backing up your files. Backups performed by the computing center should be considered your "secondary", not "primary", source of backup.

If you wish to restore a lost or damaged disk data set and are unable to restore the data set from your own backup facilities, it may be possible to restore the data set from backup facilities maintained by the computing center.

Once a day, the computing center makes tape backups of disk data sets which have been modified since the last backup. Also, once a month, all disk data sets, whether modified or not, are copied to backup tapes. The daily backup tapes (incremental backups) are kept for 42 days; the monthly backup tapes are maintained for 13 months. (Files opened for output or update at the time backup occurs are bypassed for that day's backups.)

The backup facilities maintained by the computing center are intended for use by the computing center for disaster recovery purposes. As a public service, these facilities are available to supplement your own backup facilities. You should manage your own set of backups to ensure that you will always have a permanent "good" copy of important and non-reproducible disk data sets. In particular, if you have a newly created or modified data set which is inadventently deleted or modified before the daily backups are done, recovery of the data will depend primarily on your own backup facilities.

You need to complete the "Request for Data Set Services" form to have a data set restored. This form and instructions for filling it out are available at the computing sites. After you have submitted a properly completed form to the site operator, an operator will attempt to restore your data set from system backup tapes to temporary disk (with volume serial number ASU234) within 24 hours, but special cases may take more time.

After submitting the form, you'll need to follow two steps to recover your data set:

  1. Periodically, use ISPF 3.4 to check on whether the file has been restored. On menu 3.4, fill in the name of the data set AND put ASU234 in the volume serial number field before pressing Enter. If the data set cannot be found, it means the restore has not completed yet.
  2. Once the data set has been restored, use ISPF 3.3 to move or copy the data set (or members thereof) to another data set. Be sure to include the volume serial number ASU234 on the "From" menu. (If the "To" data set does not yet exist, use ISPF 3.2 to create the "To" data set before using ISPF 3.3.)

You must perform both these tasks to recover a permanent copy of your data set. Restored data sets are placed on a temporary disk volume for at least two days and are subsequently deleted. You need to copy the data set to a permanent disk volume during this period.
