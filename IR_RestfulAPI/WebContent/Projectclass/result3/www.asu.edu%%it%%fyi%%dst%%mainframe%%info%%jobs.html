Job Classes

The system automatically determines the job class in which a job should run. This is determined by examining the resources requested by a job, namely cpu time, virtual memory, tape drives, and print lines. If any or all of these are not explicitly specified when the job is submitted, default values will be assumed for the missing categories. The table below lists the default and maximum values for each category.

            RESOURCE LIMITS FOR ALL JOBS

          |  CPU      VIRTUAL   PRINTED    TAPE
          |  TIME      MEMORY    LINES    DRIVES
  --------+---------+---------+---------+----------
  DEFAULT |   5 sec |    4M   |   10K   |    0    |
  MAXIMUM | 360 min |  256M   |  120K   |    4    |
  --------+---------+---------+---------+----------

  K = 1000 when referring to print limits
  K = 1024 bytes when referring to memory
  M = 1024K or 1024*1024 or 1,048,576 bytes when referring to memory

To specify the number of tape drives, use the JCL statement "/*TAPES n" in the jcl, where "n" is the number of tape drives needed (not the number of tape volumes nor the number of tape files).

To specify the cpu time limit, use the parameter "TIME=(min,sec)" on the JOB statement, where "min" is the number of minutes and "sec" is the number of seconds.

To specify the maximum number of lines, use the parameter "LINES=thousands" on the /*JOBPARM statement, where "thousands" is the number of thousands of print lines, i.e., "LINE=30" means 30 thousand print lines.

To specify the maximum amount of memory, use the parameter "REGION=xM" on JOB statement, where "x" is the number of megabytes. (1M is equivalent to 1024K, 2M is equivalent to 2048K, etc.)

The fewer resources you request, the better priority you will receive.
