chmod - change access permission of a file or directory

Command:
chmod [-fR] mode file_name

Summary:
The unix 'chmod' command changes the access permissions or modes of the specified files or directories. Modes determine who can read, change, or execute a file.

Often_Used_Options:
-R can be used when a directory is specified on the command line. chmod will give all the subdirectories and files under that directory the attributes specified for the directory itself.

Modes:
The mode has the form:
[who] op permission

The who value can be any combination of the following:

u --- sets user permissions.
g --- sets group permissions.
o --- sets other permissions.
a --- sets all permissions.

The op is an operator telling whether permissions should be turned on or off.

+ --- turns on a permission.
- --- turns off a permission.
= --- turns on the specified permissions.

The permission is an combination of the following:

r --- read permission. If this is turned off, you will not be able to read the file
w --- write permission. If this is turned off, you will not be able to write to the file
x --- execute permission. If this is turned off, you will not be able to execute the file

Examples:
  1. To turn off the write permission, enter:

    chmod -w file_name

  2. To turn on the execute permission, enter:

    chmod +x file_name

  3. To turn on all read, write, and execute permissions, enter:

    chmod a=rwx file_name

  4. To turn only read permissions for all, and read, write, and execute permissions for self, enter:

    chmod a=r file_name
    chmod u=rwx file_name

For addition information, enter "man chmod" at the system prompt.
