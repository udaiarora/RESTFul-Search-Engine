IT : Mac —   How to make a command in UNIX available to all users


Let say you have a user application that you wish to install under OS 10.x and make its command available so that all users can execute it from their respective terminal window. A suitable directory for installing UNIX user applications is /usr/local/apps. Since the application cammands will be stored in different sub-directories, e.g. XYZ in /usr/local/apps/XYZ-dir and ABC in /usr/local/apps/ABC-dir and so on. It is suggested that /usr/local/bin be created for storing links to the user application commands and then /usr/local/bin be added to the system path so that the application commands are available to all users.

Let's create /usr/local/apps with the command:

     mkdir /usr/local/apps
or
     sudo mkdir /usr/local/apps

Then let's say that you wish to install a user application called XYZ in /usr/local/apps/XYZ-dir. Create /usr/local/apps/XYZ-dir with the command:

     mkdir /usr/local/apps/XYZ-dir
or
     sudo mkdir /usr/local/apps/XYZ-dir

and install XYZ in into it. Now use the command:

     mkdir /usr/local/bin
or
     sudo mkdir /usr/local/bin

to create the /usr/local/bin directory. Then change your current directory to /usr/local/bin using the command:

     cd /usr/local/bin

and create a symbolic link with the command similar to:

     ln -s /usr/local/apps/XYZ-dir/XYZ XYZ
or
     sudo ln -s /usr/local/apps/XYZ-dir/XYZ XYZ

The final step involves adding /usr/local/bin to the system path. Change your current directory to /etc using the command:

     cd /etc

then using a text editor, open (or create a new one) the file called profile and add the following line to it:

     export PATH=$PATH:/usr/local/bin

Save the file and close the terminal window and then logout. Finally log back in then open a terminal window and enter the command for the user application. In this case the command is XYZ.

Caution: The information given here must be followed very carefully or you can end up corrupting your Mac system. If you are uncomfortable with the directions or the commands then please do not implement them.
