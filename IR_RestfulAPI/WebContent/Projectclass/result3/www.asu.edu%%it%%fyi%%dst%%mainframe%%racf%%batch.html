Batch Jobs

If you submit batch jobs to the MVS system from a system other than Academic MVS system, you need to include the PASSWORD parameter on the JOB statement.

The logon password is supplied by the PASSWORD parameter on the JOB statement. (Only jobs submitted from systems other than the Academic MVS system need to supply the PASSWORD parameter.) The following JOB statement illustrates the use of the PASSWORD parameter and assumes that the current logon password is APPLES:

  //AABBB  JOB  (12345),REGION=2M,TIME=1,PASSWORD=APPLES

No other JCL changes need to be made. This JOB statement would be followed by whatever other statements you would normally include in the batch job.

It is also possible to change the logon password by using a slight variation of the PASSWORD parameter on the JOB statement. In the example shown below, the current logon password APPLES is changed to ORANGES:

  //AABBB  JOB  (12345),REGION=2M,TIME=1,PASSWORD=(APPLES,ORANGES)

This latter form of the JOB statement is used if the current logon password is expired. When a batch job first enters a the system and contains this latter form of PASSWORD parameter, (assuming all other appropriate conditions are met) the system changes the logon password immediately. The system does not wait for the job to execute before changing the password.
