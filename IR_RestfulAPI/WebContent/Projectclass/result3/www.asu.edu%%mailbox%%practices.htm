Outlook Best Practices

Please see the following suggestions to help improve Outlook's performance. See also Strategies for Exchange Mailbox Management.


1. Install Service Pack 1 for Outlook 2003.

This Service Pack fixes several known bugs, including errors to a delegate when trying to open a manager's calendar. See this knowledge base article for more information: http://support.microsoft.com/default.aspx?scid=kb;en-us;872839

2. Using Cached Mode

When Cached Mode is enabled, Outlook 2003 uses a local copy of your mailbox. At the same time, Outlook maintains an online connection to a remote copy of your mailbox on the Exchange Server. This allows much faster access to messages and to attachments, because you work from the local copy instead of the server copy. For more information about Cached Exchange Mode please see this knowledge base article: http://support.microsoft.com/default.aspx?scid=kb;en-us;870926.

Also available is a webcast explaining this feature: http://support.microsoft.com/default.aspx?scid=kb;en-us;893436

To enable:

  • On the Tools menu, click E-Mail Accounts, click View or change existing e-mail accounts, and then click Next.
  • In the Outlook processes e-mail for these accounts in the following order list, click the Exchange Server e-mail account, and then click Change.
  • Under Microsoft Exchange Server, select the Use Cached Exchange Mode check box.
  • Exit and restart Microsoft Outlook.

3. Check for Outlook updates frequently

Microsoft announces updates to it's software regularly to fix known issues. Visit this webpage for updates: http://office.microsoft.com/en-us/officeupdate/default.aspx

4. Use the Auto-Archive feature.

This feature helps keep your mailbox small and lets it run more efficiently. See the knowledge base article for a detailed description of how it works and how to enable and configure: http://support.microsoft.com/default.aspx?scid=kb;en-us;830119

Auto-Archive is on by default. At scheduled intervals, a dialog box will pop up asking if you would like to Archive your older items now. Simply choose Yes. Your Contacts folder is never archived; only your other folders. The default aging periods for folders are as follows:

  • Inbox, Calendar, Tasks, Notes, Journal, Drafts: 6 months
  • Sent Items, Deleted Items: 2 months
  • Outbox: 3 months

Note: Folders you create that contain the same type of items as Inbox, Calendar, Tasks, Notes, or Journal are subject to the 6 month aging period, too.

The way Microsoft Outlook determines the age of an item varies based on the item type.

E-mail message

The received date or the last modification date/time, whichever is later.

Calendar item

The appointment, event, or meeting date or the last modification date/time, whichever is later. Recurring items are not AutoArchived.

Task

The completion date or the last modification date/time. Tasks that are not marked complete are not AutoArchived. Tasks assigned to others are archived only if marked complete.

Note

The last modification date/time.

 

5. Implement these two Calendar Option settings:

Open Outlook and from Tools > Options, set the following:

  • Free/Busy Options:
    • Under the Preferences tab, choose the Calendar Options button by Customize the Appearance of the calendar
    • Choose the Free/Busy Options... Tab
    • Set it to publish 2 months every 60 minutes (increase the frequency for a highly dynamic calendar)
  • Planner Options:
    • Under the Preferences tab, choose the Calendar Options button by Customize the Appearance of the calendar
    • Choose Planner Options... Tab
    • Uncheck all options for best performance
( See "How to turn off Planner Options feature" in these knowledge base article: http://support.microsoft.com/default.aspx?scid=kb;en-us;839862 and http://support.microsoft.com/default.aspx?scid=kb;en-us;324374 for an explanation.)

6. Have a "delegate" use the same version of Outlook as the mailbox owner

To reduce the likelihood of errors, delegates should be using the same version of Outlook as the manager. See this knowledge base article for more information: http://support.microsoft.com/default.aspx?scid=kb;en-us;829217

7. Make changes to your IP range (home)

ASU IT is recommending VPN users adopt home private IP addresses so that the range for one user doesn't overlap those of another user. You will have to configure the range of IP addresses for your router manually. These are the instructions provided from Data Comm:

Configuration setting for Home Router:

LAN IP address (default route) : 192.168.0.1
IP Subnet mask : 255.255.0.0

For DHCP address range :

If you are a faculty : write down the last four digits of your ASU ID #: xx.yy
The zz will be equal to yy+50 (zz=yy+50)

Staring IP address : 192.168.1xx.yy
Ending IP address : 192.168.1yy.zz

If you are a staff: write down your last four digits of your ASU ID #: xx.yy
The zz will be equal to 1yy+50 (zz=1yy+50)

Starting IP address : 192.168.xx.1yy
Ending IP address : 192.168.xx.zz

If you are a student : write down the last four digits of your ASU ID #: xx.yy
The zz will be equal to yy+50 (zz=yy+50)

Staring IP address : 192.168.xx.yy
Ending IP address : 192.168.yy.zz
