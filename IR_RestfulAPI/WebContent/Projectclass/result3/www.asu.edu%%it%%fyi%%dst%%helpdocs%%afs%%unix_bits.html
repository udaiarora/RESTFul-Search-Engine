Using UNIX Mode Bits in AFS to Control File-Access at File-level

In the previous document you were shown how to set up access to files at the directory level. Here you will be shown how to control "read", "write" and "execute" access priviledges to the files in a directory with appropriate pre-set directory-level access priviledges.

In UNIX, each file has three "rwx" groupings that specify the privileges for "owner", "group", and "other". Where, the 'r' stands for read, 'w' for write, and 'x' for execute and are called "UNIX mode bits". The UNIX command that can be used to display the mode bits for all the files in a directory is:

ls -l

Suppose the "ls -l" command when used on a directory displays:


     -rw-r-----   1 agjjc  agjjc  7801 Aug 13 15:07 afs.rtf
     drwxr-x---   2 agjjc  agjjc  2048 Jul 16 13:41 tips

The first character or mode bit for each line in the output displays information about the type of item ('l' for a link, 'd' for a directory, and '-' for a file), the next three mode bits display information for the "owner" grouping, the next three for the "group" grouping, and the last three for the "other" grouping.

AFS "respects" the UNIX mode bits for only the owner grouping and that means a user with appropriate AFS rights can do the following:

  1. Only read a file if its UNIX 'r' owner mode bit (first character) is 'r' and not '-'.
  2. Only write to a file if its UNIX 'r' and 'w' owner mode bits (second and third characters) are not '-'.
  3. Only execute a file if its UNIX 'r' and 'x' owner mode bits (second and fourth characters) are not '-'.
