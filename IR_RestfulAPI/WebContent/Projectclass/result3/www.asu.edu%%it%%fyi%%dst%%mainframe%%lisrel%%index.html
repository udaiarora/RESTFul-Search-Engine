IT : Mainframe MVS/TSO —   LISREL



On the MVS system, LISREL and PRELIS are executed from within SPSS as procedures. For more information, refer to the documentation for SPSS.

"Standalone" versions of LISREL and the PRELIS preprocessor are available on the Research Unix Cluster. (The vendor does not offer any versions for the Stats Unix Cluster.)

IMPORTANT USAGE NOTE FOR RUNNING PRELIS & LISREL UNDER SPSS RELEASE 4.

Under release 4, LISREL is run as a command rather than as a "user" procedure, a "userproc." There is an infelicity in the SPSS code which causes an SPSS system file obtained with the "GET FILE" command to be "NOT FOUND" by the LISREL command unless there is an intervening SPSS command, a "procedure" which does something with the data. In order to set the signal flag or switch which LISREL seems to be checking, it is sufficient to insert an SPSS "EXECUTE" command after the "GET FILE" command and before the {PRELIS/}LISREL commands.

// JOB
// EXEC SPSS
//LISFILE DD DSN=WYL.AB.CDE.SPSSFILE,DISP=SHR
GET FILE LISFILE
EXECUTE           <------ INSERT THIS COMMAND
PRELIS
    {prelis statements}
LISREL
    {lisrel statements}
FINISH.
//
