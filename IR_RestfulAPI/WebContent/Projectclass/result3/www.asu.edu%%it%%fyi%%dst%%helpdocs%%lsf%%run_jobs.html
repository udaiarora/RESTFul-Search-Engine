Are there script files that I can use for executing EQS, SAS, SPSS and Splus jobs via LSF on the batch-only servers?

Yes, there are script files available for doing just that. And it is recommended that the batch servers be used for those jobs that need large amount of system resources or for those jobs that you wish to submit and then logoff the system. Following are the commands for submitting jobs from interactive servers (Research3, Research4, or Research5) via LSF to batch-only server (Batch1) using the script files.

The syntax for an EQS job is:

For version 5.1:
runeqs QUEUE_NAME JOB_NAME

For version 5.6:
None

The output file that is created will be called "JOB_NAME.out".

The syntax for a SAS job is:

For version 8.2:
runsas QUEUE_NAME JOB_NAME

For version 6.12:
runsas_old QUEUE_NAME JOB_NAME

The output file that is created will be called "JOB_NAME.lst".

The syntax for an SPSS job is:

runspss QUEUE_NAME JOB_NAME

The output file that is created will be called "JOB_NAME.out".

The syntax for an SPLUS job is:

For version 5.1:
runsplus QUEUE_NAME JOB_NAME

For version 3.4:
runsplus_old QUEUE_NAME JOB_NAME

The output file that is created will be called "JOB_NAME.out".

Where, QUEUE_NAME can be short, medium or long.

There is available another script file which allows you to check the status of the LSF queues and your jobs.

The syntax for q is:

q
