Creating/Running uni-REXX scripts

A uni-REXX script (program) may be created with any text editor and may be executed either implicitly or explicitly. Implicit execution means you execute the script as if it were another Unix command; i.e., the name of the script is the name of the command. Explicit execution means you execute the rxx command, using the name of the script as an operand.

If you execute the REXX script implicitly, the first line of the script must be the following implicit execution string.

#!/usr/local/bin/rxx

Also, the execute permission for the file must be set. Use the chmod command to set the execute permission.

chmod u+x filename

If the REXX script is executed explicitly or is compiled then the implicit execution string is optional.

To explicitly execute the REXX script, use the rxx command with the name of the script as an operand.

rxx filename

rxx uses the following search order to locate the filename.

  • current working directory
  • directories specified in the environment variable "REXXPATH"

If the current directory contains both "filename" and "filename.rex", the command "rxx filename" will execute "filename" and not "filename.rex". To execute "filename.rex" in this situation, use "filename.rex" instead of "filename" on the command line.

A sample uni-REXX script is shown below:

#!/usr/local/bin/rxx
/* This is a sample program */
say 'Hi there - welcome'
exit

If the above code is in a file named "sample", it may be executed in either of the following ways:

sample
rxx sample

The first method assumes you've previously used "chmod" to set the file's execute attribute.
