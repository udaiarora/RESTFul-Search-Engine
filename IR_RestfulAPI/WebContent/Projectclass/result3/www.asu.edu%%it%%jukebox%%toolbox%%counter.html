CGI Tools - Counter

A counter is a small program that tracks the number of people who visit or access your webpage(s). This CGI will show on your page the number of visitors and updates itself in real time. To use the counter in its basic form use the following tag:

<img src="http://cgi.asu.edu/cgi-bin/Count.cgi?df=asuriteid.dat">

Please use your ASURITE User ID instead of asuriteid.dat in the line above. Where the counter appears on your page depends on where you place this line in your HTML. The codes in the script provide a variety of functions that you can change to enhance the appearance of the counter on your page, or even to make the counter "invisible" so that only you can access it. More information on how to do this is available at the author's example page.

Troubleshooting:

  • You want more than one counter: Simple enough. In place of asuriteid.dat, specify different things, always starting with your ASURITE id. For instance, Joe Blow could have counters for pages 1, 2, and 3, and the source for their image tags might read, respectively:

    http://cgi.asu.edu/cgi-bin/Count.cgi?df=joeblow1.dat
    http://cgi.asu.edu/cgi-bin/Count.cgi?df=joeblow2.dat
    http://cgi.asu.edu/cgi-bin/Count.cgi?df=joeblow3.dat

    Get it? You can replace the 1, 2, and 3 with the name of each page or whatever is meaningful to you. Just be sure to start with your ASURITE id so you can be guaranteed that your counter id will be unique.

  • "404 Not found" or other server error: If you're accessing the counter from a server other than www.asu.edu or www.public.asu.edu, you'll need to specify the full URL of the script: http://cgi.asu.edu/cgi-bin/Count.cgi?df=asuriteid.dat (remembering to change asuriteid.dat as directed above). In all cases, cgi.asu.edu is the preferred host.
  • "Host: [your host] is not in the auth block of the conf file": Use of the counter is restricted to ASU Web servers. If you're trying to use the counter from a non-ASU server, it just won't work. If you receive this error from a page on an official ASU Web server, please notify the Webmaster of the host name in question.
  • Your access count starts out higher than one, or seems to be advancing too rapidly: Did you remember to change asuriteid.dat to your ASURITE ID? If not, you're sharing your access counts with everyone else who forgot!
  • Your access count is freaky, generating random-ish numbers in the tens of thousands: Double-check your HTML. This usually indicates that a filename wasn't passed, which usually indicates a typo or a missing ampersand between values passed.
  • You want to "reset" your properly functioning access count: You can add a number (or other alphanumeric string) after your ASURITE ID to generate a new file, or ask the Webmaster to delete your old file (remember to include the name of the file -- the stuff after df= in the <img> tag).
  • You want other fonts, or foreign digits: Currently you can use these digit styles: A through H, abacus, codo, indian, kanji. We're planning to add some ASU-specific digit styles and we'll consider any freely available digit style you come across and want to use. Information on how to use different digit styles is available at the example page mentioned above.
