Using ANSIPRINT to print from UNIX server to local printer

Q.  Help! ansiprint is just sending printing to my screen. Others using the same PC get ansiprint to send to local printer. This is happening both at the UNIX prompt with 'ansiprint FILENAME' and within PINE (where I have set my default printer to 'print-to-ansi' or something like that).

A.  Some communication software such as Kermit, EWAN, and Exceed support "ansiprint" which allows for printing of text files to local printer, and there are others such as the telnet software that comes with Windows 95 that do not. In this case, you are using a terminal emulator (communication software) that does not support ansiprint.
