Ada - General Information

Since Ada is so resource-intensive, it's on a separate server: ai.asu.edu. You need to self-subscribe to this machine if you don't already have access to it. You can do this via the web at: 

     http://www.asu.edu/selfsub/

The Ada used at ASU is GNAT, the GNU Foundation Ada Translator. You need to set the following environment variables. You may want to add these to your .profile file: 

PATH=/usr/local/gnat/bin:$PATH
GCC_EXEC_PREFIX=/usr/local/gnat/lib/gcc-lib/sparc-sun-solaris2.5.1/2.7.2/
ADA_INCLUDE_PATH=/usr/local/gnat/adainclude
ADA_OBJECTS_PATH=/usr/local/gnat/lib/gcc-lib/sparc-sun-solaris2.5.1/2.7.2/adalib
C_INCLUDE_PATH=/usr/local/gnat/lib/gcc-lib/sparc-sun-solaris2.5.1/2.7.2/include
export PATH GCC_EXEC_PREFIX ADA_INCLUDE_PATH ADA_OBJECTS_PATH C_INCLUDE_PATH


There are two examples in /usr/local/gnat/examples that you can use to get started: "hello" and "stacks". Copy them to your AFS filespace and then follow the steps below to compile both: 

ai> gcc -c hello.adb
ai> gnatbl -o hello hello.ali

ai> gcc -c stacks.adb stackdrv.adb
ai> gnatbl -o stacks  stackdrv.ali
This will produce two executable files, "hello" and "stacks" which you can then execute, e.g.: 
ai> ./stacks
new stack is empty: passed
stack with storage is not empty: passed
stack is LIFO for 1 element: passed
stack is LIFO with 2 elements: passed
