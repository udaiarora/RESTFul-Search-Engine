WebAuth Change Log

WebAuth V2 is now in production. It is recommended that developers upgrade their WebAuth V1 enabled applications to use WebAuth V2. WebAuth V2 uses an authenticator instead of a cookie, however, the authentication mechanism remains the same. Therefore, there is no need to replace the verify application. Only the portion of your code that checks for a cookie will be changed to verify the authenticator. See the code examples for implementation examples.
