Exchange Performance Improvement

As use of e-mail at ASU has grown, so has the demand on the infrastructure responsible for providing the e-mail service. Mailbox quotas increased 500% this past year (from 50M to 250M) and e-mail volume has doubled over that same time. Information Technology has continued to expand the capacity and reliability of the Outlook/Exchange infrastructure, but there are still performance problems associated with the e-mail and calendar service. Below is a list of steps that have been taken and are yet to be taken to improve the performance of this service and to keep up with the constant growth in usage.

Increase performance, capacity and reliability of the disk storage system.

  • Jun 2004: Mirror all Exchange disk (mirrored RAID sets).
  • Jun 2004: Enable failover clustering.
  • Jul 2004: Add 2 TB (1 TB raw storage translates to 150GB usable for Exchamge mailbox space) to accomodate increased mailbox sizes.
  • Aug 2004: Add 1 TB to expand log disks (increated transactions).
  • Nov 2004: Add 2 TB to accomodate 2 new mailbox servers.
  • Feb 2005: Offline defrag for all servers (reclaimed 150GB).
  • Mar 2005: Upgrade disk controllers (NetApp 880 to NetApp 980).
  • May 2005: Add 3 TB to accomodate Exchange 2003 upgrade, West Campus server migration and increased mailbox sizes.

Upgrade existing servers and add new servers

  • Oct 2004 - Dec 2004: Increase from 4 to 7 mailbox servers and upgrade hardware on original 4 mailbox servers, 2 front-end servers and the public folder server.
  • There are now a total of 10 servers in the Exchange farm.

Analyze Exchange Infrastructure

  • Jan 2005: Bring in Microsoft Engineer for Exchange Health Check
  • Jan 2005: Bring in NetApp Engineer for Disk Subsystem check
  • Feb 2005: Bring in Microsoft Engineer for Active Directory Health Check.

Implement e-mail archiving

Jun 2005: Target implementation date. The intention is to off-load Exchange primary storage by archiving old e-mail to secondary storage. The archiving process replaces the old e-mail with a shortcut that is much smaller than the original message. The user can still retrieve the archived item transparently, however mailbox searches must include a separate search of the user's archive.

Migrate to Exchange 2003

Jul 2005: Target implementation date. Move all mailboxes from Exchange 2000 servers to Exchange 2003 servers in a swing migration.
