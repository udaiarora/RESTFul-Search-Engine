Why do I need SecuRemote?

SecuRemote is used to encrypt traffic between a workstation and a remote host. It secures the information being sent between the workstation the the remote host. The software is installed on client machines and is used to access resources behind CheckPoint firewalls and the Campus VPN. The SecuRemote software can be used from any machine anywhere.

Why does SecuRemote prompt me for a password when my machine boots?

When the workstation is started in the morning, the SecuRemote software is going out to the management server to download the latest topology. The new version of the server and client software requires that the client authenticate to get this information.

When NOT on ASU's network or on ASU Wireless Network do I need to change any settings?

There are two settings that need to be changed if the workstation is behind a firewall or a router that gives out dynamic IP addresses. The settings are Force UDP encapsulation and Support IKE over TCP. High Speed Internet users will need to enable the Force UDP encapsulation. Support IKE over TCP is a very rarely needed setting, try this setting when the UDP encapsulation does not work when coming from a corporate network.

Setting Force UDP Encapsulation
1. Click on the envelope icon with the golden key in the task bar.
2. The SecuRemote window will open. Go under the tools menu.
3. Select Advance IKE Settings...
4. Click on the box before the Force UDP Encapsulation.
5. Click the OK button to return to the main SecuRemote window.
6. Go under the File menu and select Stop VPN-1 SecuRemote.
8. Go under the Start menu to Programs to Check Point VPN-1 SecuRemote.
9. Select SecuRemote to start the SecuRemote application.


Setting Support IKE over TCP
1. Click on the envelope icon with the golden key in the task bar.
2. The SecuRemote window will open. Go under the tools menu.
3. Select Advance IKE Settings...
4. Click on the box before the Support IKE over TCP.
5. Click the OK button to return to the main SecuRemote window.
6. Go under the File menu and select Stop VPN-1 SecuRemote.
7. Go under the Start menu to Programs to Check Point VPN-1 SecuRemote.
8. Select SecuRemote to start the SecuRemote application.

Why does the window have MS San Sarif instead of my ASURITE ID?

A flaw in either the SecuRemote client or server, sometimes the name of the font is returned to the userid field instead of the userid. This issue occurs after authentication has occured and a new firewall or VPN is prompting. The userid is passed through to the next firewall but is displayed incorrectly. We have raised the issue with Check Point, for now just enter your password. The connection is secure.

How often will SecuRemote prompt for userid/password?

SecuRemote will prompt for a userid and password when the software is first launched for the day. It will prompt every time the user is going to a new resource behind a different firewall or VPN. For example, you will be prompted when the workstation is first turned on in the morning. This is to download the latest topology. If the user then goes to the HR system, they will be prompted for just their password. If the user is coming from off campus through an external ISP, the user is prompt for their userid and password when connected. They will then also be prompted whenever they are going to a different resource. For example, they would be prompted again when going into the SIS or HR system.

What Operating Systems are currently supported?

Currently only the Microsoft Windows operating system is supported, versions Win98,NT,2000,XP,ME.

The Linux version is current in BETA.

The Macintosh version that will work with ASU's configuration is current in development. A release has not been set.

How do I get SecuRemote from prompting for userid/password all the time?

A feature in SecuRemote called Auto Local Logon can be configured so that the ASURITE userid and password are not requested. The software will automatically send the appropriate ASURITE userid and password when it is requested. The Windows userid and password as well as the ASURITE userid and password are required for configuration.

    To configure Auto Local Logon:

  • Click on the envelope icon with the golden key in the task bar. The SecuRemote window will appear.
  • Go under the Passwords menu to Configure Auto Local Logon. This is where you enter the userid and password for logging onto the workstation and the userid and password used for accessing SecuRemote.
  • Click OK when finished.
  • Go under the Passwords menu to Enable Auto Local Logon." Go under the Passwords menu to Enable Auto Local Logon.
  • Go under the File menu to Stop VPN-1Securemote.
  • Restart the computer for the settings to take hold.

How do I get multiple computers behind one router connect with SecuRemote?

First the router needs to be upgraded to the latest firmware level. The manufacture web site will contain this information as well as information on how to install the firmware.

The computers need to be running the latest version of SecuRemote. That version is SecuRemote NG AI. It can be downloaded from the ASU Security web site.

SecuRemote will need to be configured with Force UDP Encapsulation enabled. See Question 3 above.

[Back to top of VPN FAQ]



What do I do when the error says that it can not connect?

When you can not connect to the ASU VPN there are a couple of reasons. Make sure that Force UDP encapsulation is enabled if the computer is behind a router on a High Speed Internet network or on the ASU Wireless Network.

Second, if a firewall is running on the computer or the router the ports for SecuRemote must be open. See Question 10 for more information.

Finally, the router must have the latest version of the fireware installed for VPN to function correctly.

[Back to top of VPN FAQ]



What ports need to be open on a firewall for SecuRemote to work?

When a firewall is running on either an external router or directly on the computer several communications ports needs to be open for SecuRemote to function properly.

The ports are TCP 256 and 264 and UDP 259 and 500.

How to use X Windows with SecuRemote?

Using X Windows SecuRemote through either the ASU Wireless network or the campus VPN, the following will need to be done to determine what the translated IP address is to issue the X Windows session.

Getting IP information for X Windows Session
Open a SSH session to general.asu.edu using your ASURITE userid and password.
At the prompt type in who. This will display the current logged on users and their corresponding IP address or DNS name.
When opening a X Windows session, open it to that IP address or DNS name.

Installation Tips

  • Disable any anti-virus software.
  • Disable any firewall software.
  • Sometimes the firewall will actually need to be removed during installation of SecuRemote.
  • Open the communications ports required by SecuRemote.
  • If running a router, update the fireware software to the latest version.
  • If behind a router or on ASU Wireless, enable the Force UDP Encapsulation.
  • Configure Auto Local Logon to not have to reply to prompts.

How do I prevent SecuRemote from launching at startup?

Configure SecuRemote Not to Launch at Startup
To prevent SecuRemote from launching at start up:
1. Select the Start menu -> Settings -> Control Panel -> Administrative Tools -> Services.
2. In the Services window, select the Check Point SecuRemote service on the right pane of the window.
3. Select the Action menu -> Properties.
4. In the Check Point Service Properties dialog box, select the General tab.
5. In the General tab, select Manual from the Startup type drop down list.
6. Click the OK button. This will return you to the Services window.
7. In the Services window, select the Check Point SecuRemote Watchdog service on the right pane of the window.
8. Select the Action menu -> Properties.
9. In the Check Point SecuRemote Watchdog Properties dialog box, select the General Tab.
10. In the General tab, select Manual from the Startup type drop down list.
11. Click the OK button. This will return you to the Services window.
12. Reboot the machine to verify that the SecuRemote client does not start automatically at boot time.
 
To start the SecuRemote client:
1. Select the Start menu -> Settings -> Control Panel -> Administrative Tools -> Services.
2. In the Services window, select the Check Point SecuRemote Watchdog service in the right pane of the window.
3. Select the Action menu -> Properties.
4. In the Check Point SecuRemote Watchdog Properties dialog box, select the General Tab.
5. In the General tab, click on the Start button. This will automatically start the Watchdog service as well as the Check Point SecuRemote Service.
6. Click the OK button. This will return you to the Services window.
7. Close the Services window.
8. If SecuRemote icon (envelope with gold key) did not show up in the task bar in the lower right hand corner of the screen, start the SecuRemote application by selecting the Start menu -> Programs -> Check Point VPN-1 SecuRemote -> SecuRemote.
