C Compiling Examples

  1. To turn on optimization (highly recommended) and get high performance from your program, use the -O option as follows:

    cc -O source.c

    For additional system dependent optimization levels and options, refer to the manuals and/or man pages.

  2. To rename your executable file, you can use the -o option as follows:

    cc prog.c -o prog.exe

    To execute (run) the program, you'd then type prog.exe instead of a.out. The naming convention of the executable enables the programmer to have and track different executables from different files in the same directory and/or account.

  3. To include routines from libraries on your own account, and also produce a listing with explicit error messages, you could use the following options:

    cc -I$HOME/dev -v -qattr -qlistopt -qsrcmsg scanfil.c

    Here is what each option does:

    • -I$HOME/dev : searches the /dev directory on your account for include files that do not start with an absolute path
    • -v : turns on verbose mode for messages printed by the compiler
    • -qattr : lists functions and include files referenced in your source code; also lists the location of the include file used. You'd expect to see your subdirectory listed, e.g., /afs/asu.edu/users/a/g/j/agjes/dev/standards.h
    • -qlistopt : Lists settings of all compiler options, even those not used on the command line
    • -qsrcmsg : Prints error messages with a pointer to the exact column where the error occurred
