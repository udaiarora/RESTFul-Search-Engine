Windows and Secure-FTP

This page will guide you through the steps you need to follow to upload, or transfer, your files from a Windows (9x, NT, or 2000) PC to your AFS account's "www" directory. You upload files using an FTP program. FTP stands for File Transfer Protocol and the letters "F-T-P" sometimes are used as a verb in describing the process of uploading such files. Thus, when someone says you "need to FTP your files to the server," they mean you will be uploading them to a server by means of an FTP application.

This tutorial describes the method of uploading using a type of FTP software known as Secure-FTP sometimes referred to by the acronym SFTP. The difference between "regular" FTP and Secure-FTP is that all communication and files that are passed between your local computer and the remote server are protected by encryption with Secure-FTP while they are not protected in any manner with regular FTP.

Secure-FTP is a component of a program called "Secure Shell" or SSH-Client which has features similar to Telnet but offers the same encryption protection that Secure-FTP offers. Secure Shell (and Secure-FTP) can be downloaded from Information Technology at from within the IT Computer Security Website.

Instructions for using Secure-FTP to upload or transfer your files:

Note: these instructions were created using SSH Secure Shell SFTP version 3.1.0.

1. First, click on Start Menu > Programs > SSH Secure Shell > Secure File Transfer Client, see graphic below. The Secure-FTP program will open.

2. Select File > Connect. A Connect to Remote Host dialog box will appear.

3. In the Connect to Remot Host dialog box, fill in the name of the server to which you want to upload your pages (most users will want to upload to general.asu.edu) and your ASURITE UserID. See graphic below. Click Connect and enter your password when prompted.

4. You may be prompted with a warning dialog box titled Host Identification, if you are prompted, click Yes to answer the question: "Do you want to save the new host key to the local database?". The host key is what makes encryption of the communication between your local computer and the server possible.

5. If you entered the information correctly and clicked the Connect button, you should see the Connection Window after a few moments. See graphic below. The connection window gives you information on your information on your current remote directory (your AFS webspace on the server general.asu.edu), a listing of the files in that space, and buttons to perform basic file transfers. The animation in the graphic below points out the basic features of the Connection Window. If you see "www" in a list of files in the box to the left, you should double click on the "www" so your window will match the one below.

The Connection Window has two main buttons and an additional set of three smaller settings. The Up Arrow button allows you to upload files from a disk or hard drive you choose to the space indicated. The Down Arrow button allows you to download selected files from a chosen directory in web space to your disk or hard drive. You should make sure that the method of transfer is set to Automatic.

5. Now that you're ready, you you can upload your files to your web space. Here's how to upload a single file:

  1. Make sure you are in the web space directory you want to upload the file to.
  2. Click on the Up Arrow, you will receive an upload dialog box (see graphic below).
  3. Select the file from your local (hard drive or floppy disk) drive.
  4. Finally, click Upload to transfer the file.

Congratulations!

You now know how to do basic uploading of your files to your web space using SFTP. You'll want to review the other options in SFTP's menus like deleting, renaming, and performing other operations on files you'll upload in the future. As you become more comfortable with the use of SFTP, your experience will grow.
