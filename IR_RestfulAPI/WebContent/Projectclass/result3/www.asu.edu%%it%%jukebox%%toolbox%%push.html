CGI Tools - Location Push

This CGI script will allow you to select a web page address or file through a pull down menu and go to it by using a submit button. The form must have the following lines to work.

<form method="post" action="http://cgi.asu.edu/cgi-bin/loc.cgi">
This calls the correct CGI script.

<select name="loc">
<option value="http://full.url.here">
</select>
The select tag allows you to have a pull down menu. The option tag allows you to choose different locations. The value of the different options must be a full URL path to a page or file.

<input type="submit" value="Go">
This ends the form and submits the choice to the CGI script.

Here's a working example of this CGI:

Here's the raw HTML code for this form:

<form method="post" action="http://cgi.asu.edu/cgi-bin/loc.cgi"> <select name="loc"> <option value="http://www.asu.edu/">ASU Web Entrance Page <option value="http://www.asu.edu/jukebox/">ASU Web Jukebox </select> <input type="submit" value="Take me There!" name="submit"> </form>

Troubleshooting:

  • "404 Not found" or other server error: If you're accessing the script from a server other than www.asu.edu or www.public.asu.edu, you'll need to specify the full URL of the script: http://cgi.asu.edu/cgi-bin/loc.cgi. In all cases, cgi.asu.edu is the preferred host.
  • "Host: [your host] is not in the auth block of the conf file": Use of this script is restricted to ASU Web servers. If you're trying to use the script from a non-ASU server, it just won't work. If you receive this error from a page on an official ASU Web server, please notify the Webmaster of the host name in question.
  • You're on the right server, but you get a "page error": Usually, this happens when the URL you've specified in the "loc" form field doesn't exist. Double-check the URL, and remember, the ASU Web servers are case-sensitive!
