C Compiler, Syntax, and Options

The cc compiler processes C source code files, object code files, or assembler source files. If the "-c" option is not specified (see below), C compiles and loads the source code and generates an executable file.

C command line syntax summary (for compiling, linking & loading):

cc [-options] filename

Some of the more common compiler options are discussed below. If you do not specify particular options, C uses a set of defaults. Options may be listed in any order following the cc command. The file name is mandatory and it must end in ".c" for files containing C code or ".o" for object files (already compiled files). Here is a partial list of some commonly used C options: 

-c         Compile only; do not call the loader
-g         Include debug tables in executable image
-I<dir>    Search in directory for include files that do 
           not start with an absolute path
-o<filename>   Name the executable file instead of a.out
-O         Optimize code generated by the compiler
-p         Generate simple profiling support code
-pg        Generate profiling support code. Provides more extensive
           profiling than -p
-v         Display verbose information on the compiler's progress
-w         Suppress informational, language-level, and warnings
-qattr     Produce attribute listing of referenced functions and 
           include files
-qlistopt  List compiler option settings in listing (.lst) file
-qsrcmsg   List source code lines with pointers to the specific 
           column where error(s) occurred
file.c     File containing 'C' source code
file.o     File containing object code
