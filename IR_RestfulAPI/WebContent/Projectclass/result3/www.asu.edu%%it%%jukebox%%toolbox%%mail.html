CGI Tools - Mail

This Java Servlet will process any form on a page and email the data to a specified valid e-mail address. The form must have the following lines to work:

<form method="post" action="https://sec.was.asu.edu/pubtools/Mail">
This calls the correct servlet.
<input type="hidden" name="sendto" value="youremail@asu.edu">
This tells the script where to mail the form data.
OR
<input type="hidden" name="sendto" value="youremail@asu.edu,yourotheremail@asu.edu">
If you want to send the message to multiple recipients.
<input type="hidden" name="subject" value="Your Subject">
This tells the script the Subject heading in the E-mail.
<input type="hidden" name="location" value="http://www.public.asu.edu/~<yourasurite>/">
This tells the script which page to go to after the submit button is pressed. This must be a full URL with the "http://" in it.

Here's a working example of this CGI:

Name:

E-Mail Address:

Enter any suggestions or comments below:

Here's the raw HTML code for this form:

<form method="post" action="https://sec.was.asu.edu/pubtools/Mail">
<input type="hidden" name="sendto" value="webmaster@asu.edu">
<input type="hidden" name="subject" value="Public CGI Test">
<input type="hidden" name="location" value="http://www.asu.edu/it/jukebox/toolbox/mail.html">
Name:
<input size=30 name="name">
<p>E-Mail Address:
<input size=27 name="email">
<p>Enter any suggestions or comments below:<br>
<textarea name="comments" rows=8 cols=45 wrap=Physical></textarea>
<p>
<input type="submit" value="Mail" name="submit">
<input type="reset" value="Start Over" name="reset">
</form>

Optional Parameters

You can add the following optional parameters in your HTML anywhere between the <form ...> and </form> html tags.

  • set_email_from_field: This option changes the resulting email's "from address" from null@asu.edu to the email address of the person submitting the form (if they submit a valid email address with the form). This option is helpful for forms that process a large number of questions that require a reply. The code you will need is below. Both parameters listed below are required for this functionality to work.

<input type="hidden" name="set_email_from_field" value="true"> <input size="25" name="email_address">

  • sendto_is_asu: This option allows form authors to replace the "normal" email address(es) required by the "sendto" field with a truncated address by dropping the @asu.edu part of the address. If you set this option, the "sendto" address must be an ASU email address (the Mail servlet will append @asu.edu automatically). This option can prevent "SPAM-bots" from stealing your email address from your HTML code and adding your address to their SPAM email-lists.

    The below code snippet will send messages to webmaster@asu.edu. Replace "webmaster" with your ASU email address minus the @asu.edu part of the address, i.e., for jane.student@asu.edu, replace "webmaster" with "jane.student". Both parameters listed below are required for this functionality to work.

<input type="hidden" name="sendto" value="webmaster"> <input type="hidden" name="sendto_is_asu" value="true">

Troubleshooting:

  • "404 Not found" or other server error: Double check that your form is using the full URL to the servlet: https://sec.was.asu.edu/pubtools/Mail. Also, check that the URL or web address you set for the "location" parameter exists.
  • "Host: [your host] is not in the auth block of the conf file": Use of this script is restricted to ASU Web servers. If you're trying to use the script from a non-ASU server, it just won't work. If you receive this error from a page on an official ASU Web server, please notify the Webmaster of the host name in question.
  • You're on the right server, but you get a "page error": Usually, this happens when the URL you've specified in the "location" form field doesn't exist. Double-check the URL, and remember, the ASU Web servers are case-sensitive!
