uni-XEDIT Commands

Editing commands can be typed in two general areas: on the command line and in the prefix area. The command line is usually at the bottom of the screen. (You can change the location of where the command line appears.) The prefix area is usually the five columns immediately to the left of each data line on the screen. (You can change the location of the prefix area or can remove the prefix area entirely from the screen.)

Commands which can be typed on the command line are called line commands. Commands which can be typed in the prefix area are called prefix commands.

Refer to the reference manual for more comprehensive descriptions of the commands and for descriptions of commands not listed below.

Prefix Commands

a
Adds one line
an
Adds 'n' lines
c
Copies one line
cc
Copies a block of lines
d
Deletes one line
dd
Deletes a block of lines
f
Indicates the line after which a copy (c) or a move (m) is to be inserted
p
Indicates the line before which a copy (c) or a move (m) is to be inserted
i
Inserts one line
in
Inserts 'n' lines
m
Moves one line
mm
Moves a block of lines
"n
Replicates one line 'n' times
""
Replicates a block of lines

Line Commands

change /string-1/string-2/ xx yy
Searches for a string of characters and changes it to a different string, where "string-1" is a search string and "string-2" is the replacement string. "xx" specifies the number of lines in which the changes should occur relative to the current line, and "yy" specifies the number of occurrences of "string-1" to be changed in each line. "xx" can be an asterisk (*) to represent all lines from the current line to the end of file, and "yy" can be also an asterisk to represent all occurrences in a line. "xx" can be prefixed with a minus sign to work backwards from the current line to the beginning of the file.

In the following sample commands, the first command changes the first occurrence of "abc" on the next 10 lines including the current line. The second command changes all occurrence of "abc" on all lines from the current line to the end of the file:
ch/abc/mno/ 10 1
ch/abc/mno/ * *
/text
Searches forward in the file e.g., "/sas" searches for the string "sas".
bottom
Scrolls to the bottom of the file.
top
Scrolls to the beginning of the file.
down n
Advances 'n' lines towards end of the file.
up n
Advances 'n' lines towards top of the file.
file
Writes the current file to disk and exists the editor.
ffile
Overwrites any other file with the same name even if that file changed since your edit session began.
save [filename]
Writes the current file to disk. If a filename is included on the command, the data will be written to the specified file instead of the current file.
ssave [filename]
Overwrites any other file with the same name even if that file changed since your edit session. If no filename is used, the current filename is used.
quit
Terminates the edit session if no changes have been made to the file since the last SAVE.
qquit
Terminates the edit session even if changes have not been saved.
left n
Shifts 'n' columns to the left.
right n
Shifts 'n' columns to the right.
get filename
Retrieves a file and places after the current line.
input string
Inserts a single line of text into the file or puts uni-XEDIT in input mode. In the following sample commands, the first command inserts a new line containing "string" immediately after the current line. The second command puts uni-XEDIT into input mode which permits multiple lines to be inserted into the file. To exit input mode, press the ENTER key twice.
input string
input
help
Displays the help file.
recover n
Recovers 'n' number of deleted lines (last ones are recovered first). "rec *" recovers all the deleted lines.
shift right n m
Moves data 'n' columns to the right on 'm' lines. For example,
shift right 5 *
moves data five columns to the right on all (*) lines. "r" can be replaced with "left" if you wish to move the data to the left.
cfirst
Shifts the column pointer left to the leftmost column that you can edit.
zone x y
Limits the column scanning for target searches between starting (left) and ending (right) columns. For example,
zone 5 *
limits scanning between column 5 and the rightmost column and includes the column 5 and the rightmost column.
:n
Identifies 'n' as the current line.
?
Displays the last command.
