Windows and Secure-Shell (SSH)

What is SSH?

SSH Secure Shell is the industry standard for remote access to host machines. SSH solves the most important security problem on the internet, securing userids and passwords. ASU provides clients for Windows 9x, Windows NT, Windows ME, Windows 2000/XP, Macintosh, and several UNIX flavors. Clients are available for download for ASU affiliates through the Information Technology Communications Consulting website.

Starting SSH on Windows

Start > Program Files > SSH Secure Shell > Secure Shell Client. See image below.

After the SSH Secure Shell program window opens, click File > Connect. This will open a "Connect to Remote Host" dialog box. In the text-field labeled "Host Name:" type "general.asu.edu". In the field labeled "User Name:" type your ASURITE UserID. See image below.

Click the Connect button. You will be prompted for your ASURITE password. Type in your password and click the OK button or hit Return. After your password is accepted, you should see a prompt similar to the image below. You are now logged in to a "console" on general.asu.edu. At this console you can type unix commands or edit files in your favorite unix text editor.

I'm connected to general.asu.edu, now what?

The general purpose server that you're connected to is a UNIX based system and you have to communicate with it through keyboard commands. Following are a few commands you'll need to make your secured page(s).

  • 'ls' Lists the contents of a directory.
  • 'ls -al' Lists the contents of a directory along with hidden files and other useful information.
  • 'pwd' Lists the path to the directory you're currently in.
  • 'mkdir secretstuff' Makes a new directory, in this case the directory is named 'secretstuff'.
  • 'cd secretstuff' Move to the secretstuff directory.
  • 'cd ..' Move up one directory.
