Syntax Summary

f77 [-options] filename
For compiling, linking and loading
-c
Compile only; do not call the loader.
-C
Perform run-time checking of array bounds and character substring expressions.
-g
Produce debug information.
-I<dir>
Search in directory for INCLUDE files that do not start with an absolute path.
-o<filename>
Name the executable file instead of a.out.
-O
Optimize code generated by the compiler.
-p
Generate simple profiling support code.
-pg
Generate profiling support code. Provides more extensive profiling than -p.
-v
Display verbose information on the compiler's progress.
-w
Suppress informational, language-level, and warnings.
file.f
File containing FORTRAN-77 source code file.
file.o
File containing object code.
a.out
To execute the program, after successful compile, link and load

f77 Compiler

The f77 compiler conforms to the American National Standards Institute (ANSI) standard X3.9-1978, often called FORTRAN 77. f77 processes FORTRAN source code files, object code files, or assembler source files. If the "-c" option is not specified (see below) f77 compiles and loads the source code and generates an executable file.

The format of the f77 command line is:

f77 [options] file

Some of the more common compiler options are discussed below. If you do not specify particular options, f77 uses a set of defaults. Options may be listed in any order following the f77 command. The file name is mandatory and it must end in ".f" for files containing FORTRAN-77 code or ".o" for object files (already compiled files).

Program Execution

The simplest method of program execution is the "compile, load and go" sequence. Here is a FORTRAN example:
f77 source.f
a.out

In these example, the f77 commands compiles and loads the programs "source.f" and automatically generate the executable file "a.out". To execute "a.out", simply type the word "a.out" at the UNIX prompt. Your program is then off-and-running.

Program Input and Output

A FORTRAN-77 program can open files with any unit number in the range of 1 to 99. To write to or read from a file, one can open the file with a FORTRAN OPEN statement as follows:
OPEN(UNIT=33, FILE='example.in', STATUS='OLD') <-- to read OPEN(UNIT=35, FILE='example.out', STATUS='NEW') <-- to write.

Once the program is done reading or writing, the CLOSE statement can be issued to close the file. For example :

CLOSE(UNIT=35) <-- to close the "example.out" file

If any unit number is used to read or write and not specified in any OPEN statement, the default file name will be "fort.UNIT"". For example, if UNIT=66 is not associated by an OPEN statement with any file name, then, by default, the following statement will create and write to a new file "fort.66" in the present directory:

WRITE(66,...

By default, the unit number for standart input (stdin, SYSIN, read from the keyboard) is 5 and the standard output (stdout, SYSOUT, write to the terminal) is 6. It is wise not to use those unit numbers in any other way. Furthermore, UNIX FORTRAN-77 allows the user to redirect the default input (UNIT=5) and/or output (UNIT=6), as any other UNIX command. This is very useful in batch mode, where one can supply the keyboard input (stdin) via a file, and/or capture the screen output (stdout) into a file. For example :

a.out < param.in > result.out

In the above program, any "READ(5,..." statement will read from the file "param.in", and any "WRITE(6,..." or "PRINT ,..." statement will write to the file "result.out".

Examples

Most of the compilers do optimization by default (except IBM). To ensure optimization and high performance of your program use the "-O" option as follows:

f77 -O source.f

Depending on the program, it may result in more than 2 times faster execution. For more system dependent optimization levels and options refer to the manuals and/or man pages.

To be able to rename your executable file you can use the "-o" option as follows:

f77 prog.f -o prog.exe

To execute (run) the program you can type "prog.exe" instead of "a.out". The naming convention of the executable enables the programmer to have and keep track of different executables from different files in the same directory and/or account.

To link libraries such as IMSL, please refer to:

The Online UNIX IMSL help

Some of the compilers (linkage and loaders) are very cautious, thus if you have the same name used for two different routines, they will not load the program nor produce an executable.

RS/6000 "f77" and "xlf" Compilers

Note that the f77 compiler from IBM is not equivalent to the standard UNIX f77 compiler that is often found on other workstations. IBM has aliased the name f77 to their own FORTRAN compiler, which is called xlf. Many of the functions and options of the two compilers are identical. In fact, one can compile FORTRAN code with either the f77 or xlf command. Because of this aliasing there is no "man" page for f77. Instead, you must use the command "man xlf" to get man page information about the f77 compiler.
