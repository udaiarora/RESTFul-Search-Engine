Secure Authentication for ASU Web Applications


Overview

WebAuth is a secure, centralized authentication system for use by ASU web applications. WebAuth provides application developers with a means of authenticating users of web applications using ASU's central Kerberos authentication system. It also provides users with a single sign-on environment, enabling users to sign-on to the ASU Portal and move seamlessly between WebAuth enabled applications, such as ASU Interactive and ASU e-mail.

WebAuth enabled applications redirect users to the secure WebAuth login page. This page prompts the user for their ASURITE UserID and password, which is authenticated using Kerberos. The WebAuth servers then appends an authenticator string on the URL redirecting the user back to the original application.

Getting Started

  1. Read How it Works - The first step for developers wishing to implement WebAuth in their applications is to familiarize themselves with the developers documentation.
  2. Review WebAuth Policies - Developers should read, understand, and comply with all ASU computing policies. The WebAuth policies page illustrates how ASU's computing policies apply to the use of WebAuth.
  3. Join WebAuth Listserv - Join the WebAuth listserv to communicate with and get advice from the WebAuth community.
  4. Register Your Application - Once developers have an understanding of how the system works, they can register their application server with WebAuth.
  5. Review Implementation Documentation - Developers should thoroughly review the WebAuth Implementation Documentation before implementing.
  6. Download WebAuth Client - Finally, developers should download the latest verify client for their platform of choice.
