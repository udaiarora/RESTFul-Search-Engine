Are there script files for submitting FORTRAN77 jobs from the interactive servers to the batch-only server?

Yes, there are script files available for doing just that. And it is recommended that the batch server be used for those jobs that need large amount of system resources or for those jobs that you wish to submit and then logoff the system. Following are the commands for submitting jobs from interactive servers (Research3, Research4, or Research5) to batch-only server (Batch1) via LSF using script files.

The syntax for executing a FORTRAN77 job is:

runf77 QUEUE_NAME jobname.f

Where, QUEUE_NAME can be short, medium or long.

There is available another script file which allows you to check the status of the LSF queues and your jobs.

The syntax for q is:

q

Executing 'runf77' creates a standard error and out file called 'f77job.o.e' and this should be checked for any error messages.

For further assistance, send an email message to "unix-q@asu.edu".
