Are there script files for submitting LISREL and PRELIS jobs to the batch-only servers?

Yes, there are script files available for doing just that. And it is recommended that the batch servers be used for those jobs that need large amount of system resources or for those jobs that you wish to submit and then logoff the system. Following are the commands for submitting jobs from interactive servers (Research3, Research4, or Research5) via LSF to batch-only server (Batch1) using the script files.

The syntax for executing a LISREL job is:

runlisrel QUEUE_NAME JOB_NAME

To increase the size of the workspace enter:

runlisrel QUEUE_NAME JOB_NAME WORKSPACE_SIZE

The output file that is created will be called "JOB_NAME.out". Furthermore WORKSPACE_SIZE is the size of the workspace being requested in Kilobytes. It is strictly a numerical value that is based on the requirements of your job. For example, if your job requests atleast 13259752 bytes of workspace then that would translate into atleast 13259752/1024=12949 KB.

Whereas, QUEUE_NAME can be short, medium or long.

The syntax for runprelis is:

runprelis QUEUE_NAME JOB_NAME

The output file that is created will be called "JOB_NAME.out".

Whereas, QUEUE_NAME can be short, medium or long.

There is available another script file which allows you to check the status of the LSF queues and your jobs.

The syntax for q is:

q
